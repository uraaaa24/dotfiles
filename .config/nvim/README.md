# Neovim

自分用の Neovim 操作メモです。
「何を押せばいいか」と「押したあとにどう戻るか」が分かるように書いています。

## 最初に覚えること

Neovim は主に「ノーマルモード」と「挿入モード」を行き来して使います。

| キー | 何が起きるか |
| --- | --- |
| `i` | カーソルの前から文字入力を始める |
| `a` | カーソルの後ろから文字入力を始める |
| `Esc` | 文字入力をやめてノーマルモードに戻る |
| `:` | コマンドを入力する |
| `v` | 範囲選択を始める |
| `V` | 行単位で範囲選択を始める |

基本の流れはこれです。

1. `i` で入力する
2. 書き終わったら `Esc` でノーマルモードに戻る
3. `:w` で保存する
4. `:q` で閉じる

## 保存と終了

保存や終了のコマンドは、ノーマルモードで `:` を押してから入力します。

| コマンド | 何が起きるか |
| --- | --- |
| `:w` | 今開いているファイルを保存する |
| `:q` | 今のウィンドウを閉じる |
| `:wq` | 保存して閉じる |
| `:q!` | 保存せずに閉じる |
| `:qa` | すべてのウィンドウを閉じる |
| `:qa!` | 保存せずにすべて閉じる |
| `ZZ` | 保存して閉じる |
| `ZQ` | 保存せずに閉じる |

よく使うのは `:w`、`:q`、`:wq`、`:q!` です。

例:

```vim
:w
```

と入力して `Enter` を押すと保存されます。

保存せずに閉じたいときは、次のようにします。

```vim
:q!
```

## ファイルを開く

| コマンド・キー | 何が起きるか |
| --- | --- |
| `:e <path>` | 指定したファイルを開く |
| `:enew` | 空の新しいバッファを開く |
| `<leader>ff` | ファイルを検索して開く |
| `<leader>fb` | すでに開いているバッファから選ぶ |
| `<leader>fg` | プロジェクト内の文字列を検索する |
| `<leader>fh` | ヘルプを検索する |

この設定では `<leader>` は `Space` です。
たとえば `<leader>ff` は、`Space` を押してから `ff` を押します。

ファイル検索を開いたあとは、探したいファイル名を入力して、候補を選んで `Enter` で開きます。
キャンセルしたいときは `Esc` です。

## バッファ

Neovim では、開いているファイルのことを「バッファ」と呼びます。

| コマンド | 何が起きるか |
| --- | --- |
| `:ls` | 開いているバッファ一覧を見る |
| `:b <number>` | 指定した番号のバッファへ移動する |
| `:bnext` | 次のバッファへ移動する |
| `:bprevious` | 前のバッファへ移動する |
| `:bd` | 今のバッファを閉じる |
| `:bd!` | 保存確認を無視して今のバッファを閉じる |

普段は `<leader>fb` で開いているバッファを選ぶのが分かりやすいです。

## サイドバー、ファイルツリー

この設定では Neo-tree を使っています。
左側にファイルツリーを出して、プロジェクト内のファイルを見たり開いたりできます。
ファイル種別のアイコンには Nerd Font が必要です。Ghostty では `HackGen Console NF` / `Hack Nerd Font` を使う設定にしています。

| キー | 何が起きるか |
| --- | --- |
| `\` | 今開いているファイルの場所を Neo-tree で表示する |

Neo-tree が開いたら、左側のサイドバーでファイルを選べます。
もしカーソルが編集画面側にある場合は、左側のサイドバーに移動します。
エディタ本体に戻るときは、ウィンドウ移動のキーを使います。

| キー | 何が起きるか |
| --- | --- |
| `<C-w>h` | 左のウィンドウへ移動する |
| `<C-w>l` | 右のウィンドウへ移動する |
| `<C-w>j` | 下のウィンドウへ移動する |
| `<C-w>k` | 上のウィンドウへ移動する |

`<C-w>h` は `Ctrl` を押しながら `w` を押して、そのあと `h` を押す、という意味です。

よくある動き:

1. `\` でサイドバーを開く
2. `<C-w>h` で左のサイドバーへ移動する
3. `j` / `k` でファイルを選ぶ
4. `Enter` でファイルを開く
5. `<C-w>l` で右の編集画面へ戻る

Neo-tree の中で使うキーです。

| キー | 何が起きるか |
| --- | --- |
| `Enter` | 選んだファイル、またはディレクトリを開く |
| `a` | ファイル、またはディレクトリを追加する |
| `d` | 選んだファイル、またはディレクトリを削除する |
| `r` | 選んだファイル、またはディレクトリの名前を変更する |
| `q` | Neo-tree を閉じる |
| `\` | Neo-tree のウィンドウを閉じる |

ディレクトリを作りたいときは、`a` を押して `src/components/` のように最後を `/` にします。
tmux の別 pane や shell でファイルを作成・削除した場合も、Neo-tree は filesystem watcher で自動更新します。
Git の状態表示は VS Code の Explorer decoration に近い見た目にしています。ファイルは右端に `M`、`A`、`D`、`U` などの状態文字を出し、変更を含むディレクトリは `●` を出します。変更済みは黄色、追加は落ち着いた緑、未追跡は明るい緑、削除は赤茶色です。

## ターミナルとの使い分け

この dotfiles では、tmux 側にも leader があります。
Neovim の `<leader>` とは別物です。

| 場所 | leader | 役割 |
| --- | --- | --- |
| tmux | `Ctrl-q` | shell の split、pane 移動、zoom、session 維持 |
| Neovim | `Space` | ファイル検索、Git、LSP、フォーマット |

tmux の split は、Neovim の外側で画面を分けます。
たとえば左で Neovim、右で `pnpm dev` や `git status` を動かしたいときに使います。

| キー | 何が起きるか |
| --- | --- |
| `Ctrl-q r` | 右に split を作る |
| `Ctrl-q d` | 下に split を作る |
| `Ctrl-q h` | 左の pane へ移動する |
| `Ctrl-q j` | 下の pane へ移動する |
| `Ctrl-q k` | 上の pane へ移動する |
| `Ctrl-q l` | 右の pane へ移動する |
| `Ctrl-q z` | 今の pane を拡大、元に戻す |
| `Ctrl-q x` | 今の pane を閉じる |
| `Ctrl-q D` | tmux から detach する |

Neovim のウィンドウ分割は、Neovim の中だけでファイルを並べたいときに使います。

| 目的 | 使うもの |
| --- | --- |
| エディタと開発サーバーを並べる | tmux split |
| Neovim と shell を並べる | tmux split |
| 2つのコードファイルを並べる | Neovim の `:vsplit` |
| Neo-tree と編集画面を行き来する | Neovim の `<C-w>h` / `<C-w>l` |

今の Neovim 設定には、任意コマンドを開く汎用 floating terminal は入れていません。
LazyGit は floating window で開きますが、shell、開発サーバー、AI CLI などは tmux pane で開く想定です。

## ウィンドウ分割

1つの画面を左右や上下に分けて、複数のファイルを同時に見られます。

| コマンド・キー | 何が起きるか |
| --- | --- |
| `:split` | 画面を上下に分割する |
| `:vsplit` | 画面を左右に分割する |
| `<C-w>h` | 左のウィンドウへ移動する |
| `<C-w>j` | 下のウィンドウへ移動する |
| `<C-w>k` | 上のウィンドウへ移動する |
| `<C-w>l` | 右のウィンドウへ移動する |
| `<C-w>q` | 今いるウィンドウを閉じる |

サイドバーと編集画面の行き来にも、この `<C-w>` 系のキーを使います。
shell や開発サーバーを並べたいときは、tmux の `Ctrl-q r` や `Ctrl-q d` を使います。

| キー | 何が起きるか |
| --- | --- |
| `u` | 元に戻す |
| `<C-r>` | やり直す |
| `yy` | 今の行をコピーする |
| `dd` | 今の行を切り取る |
| `p` | カーソルの後ろに貼り付ける |
| `P` | カーソルの前に貼り付ける |

範囲を選んでから `y` でコピー、`d` で切り取りもできます。

## 検索

| キー | 何が起きるか |
| --- | --- |
| `/` | 今のファイル内を検索する |
| `n` | 次の検索結果へ移動する |
| `N` | 前の検索結果へ移動する |
| `<leader>fg` | プロジェクト全体から文字列を検索する |

ファイル内検索の例:

```vim
/function
```

`function` を探します。検索をやめたいときは `Esc` です。

## 開発でよく使う流れ

普段の開発では、次の操作を覚えておくとだいたい困りません。

| やりたいこと | 使うキー・コマンド |
| --- | --- |
| ファイルを探す | `<leader>ff` |
| 開いているファイルを切り替える | `<leader>fb` |
| プロジェクト全体から文字を探す | `<leader>fg` |
| サイドバーを開く | `\` |
| 今のファイルを保存する | `:w` |
| 今のファイルをフォーマットする | `<leader>f` |
| 定義へ移動する | `gd` |
| エラーの詳細を見る | `<leader>e` |
| 次のエラーへ移動する | `]d` |
| 前のエラーへ移動する | `[d` |
| Git の差分を見る | `<leader>hp` |
| LazyGit を開く | `:LazyGit` または `<leader>lg` |

`<leader>` は `Space` です。
たとえば `<leader>fg` は、`Space` を押してから `fg` を押します。

### よくある作業手順

1. `<leader>ff` で編集したいファイルを開く
2. `i` で入力する
3. `Esc` でノーマルモードに戻る
4. `:w` で保存する
5. `<leader>f` で必要ならフォーマットする
6. `<leader>hp` で今の変更差分を確認する
7. `:LazyGit` または `<leader>lg` で commit や push をする

### LazyGit

LazyGit は Neovim の中から使える Git UI です。
commit、push、pull、branch 切り替え、stage などをまとめて操作できます。

起動は次が確実です。

```vim
:LazyGit
```

ショートカットで開く場合は `<leader>lg` です。
`Space` を押してから `lg` を押します。

LazyGit の中では、基本的に画面下や右側に出るヘルプ表示に従います。
よく使う操作は次です。

| キー | 何が起きるか |
| --- | --- |
| `j` / `k` | 上下に移動する |
| `h` / `l` | パネルを移動する |
| `Space` | 選択中のファイルや hunk を stage / unstage する |
| `c` | commit する |
| `P` | push する |
| `p` | pull する |
| `q` | LazyGit を閉じる、または前の画面へ戻る |
| `?` | ヘルプを見る |

commit までの例:

1. `:LazyGit` または `<leader>lg` で開く
2. Files パネルで変更ファイルを選ぶ
3. `Space` で stage する
4. `c` で commit message を入力する
5. `P` で push する
6. `q` で Neovim に戻る

### Git 差分を見る

エディタ上で軽く確認するなら gitsigns、まとめて操作するなら LazyGit を使います。

| キー | 何が起きるか |
| --- | --- |
| `]c` | 次の変更箇所へ移動する |
| `[c` | 前の変更箇所へ移動する |
| `<leader>hp` | 今の hunk の差分を見る |
| `<leader>hs` | 今の hunk を stage する |
| `<leader>hr` | 今の hunk を戻す |
| `<leader>hS` | 今のバッファ全体を stage する |
| `<leader>hR` | 今のバッファ全体の変更を戻す |
| `<leader>hb` | 今の行の blame を見る |
| `<leader>hd` | index との差分を見る |
| `<leader>hD` | 最後の commit との差分を見る |

`<leader>hr` と `<leader>hR` は変更を戻す操作です。
押す前に `<leader>hp` や LazyGit で差分を確認してください。

### コードを読む

LSP が動いているファイルでは、定義ジャンプや参照検索が使えます。

| キー | 何が起きるか |
| --- | --- |
| `gd` | 定義へ移動する |
| `K` | カーソル下の説明を見る |
| `gi` | 実装へ移動する |
| `gr` | 参照箇所を見る |
| `<leader>rn` | 名前をまとめて変更する |
| `<leader>ca` | コードアクションを開く |

### エラーや警告を見る

| キー | 何が起きるか |
| --- | --- |
| `]d` | 次のエラー、警告へ移動する |
| `[d` | 前のエラー、警告へ移動する |
| `<leader>e` | エラー、警告の詳細を表示する |
| `<leader>q` | エラー、警告一覧を location list に入れる |

まず `]d` で問題箇所へ移動し、内容が分かりにくいときは `<leader>e` を押します。

### フォーマット

保存時の自動フォーマットは無効です。
必要なときだけ手動で実行します。

```vim
<leader>f
```

フロントエンド系では `biome`、`prettierd`、`prettier` の順で使います。

## 補完

入力中に候補が出ます。補完には `nvim-cmp` を使っています。

| キー | 何が起きるか |
| --- | --- |
| `<C-Space>` | 補完メニューを開く |
| `Enter` | 選んでいる候補を確定する |
| `Tab` | 次の候補へ移動する、またはスニペット内を進む |
| `Shift-Tab` | 前の候補へ移動する、またはスニペット内を戻る |
| `<C-b>` | 補完説明を上へスクロールする |
| `<C-f>` | 補完説明を下へスクロールする |
| `<C-e>` | 補完メニューを閉じる |

候補が邪魔なときは `<C-e>` で閉じられます。

## フォーマット

保存時の自動フォーマットは無効にしています。
自分で整えたいときだけ、ノーマルモードで次を押します。

```vim
<leader>f
```

対応しているファイルでは、次の順番でフォーマッタを探します。

1. `biome`
2. `prettierd`
3. `prettier`

主な対象は JavaScript、TypeScript、HTML、CSS、JSON、Markdown、Vue、Svelte、Astro、YAML などです。

## LSP

LSP は、定義ジャンプ、エラー表示、リネーム、コードアクションなどをしてくれる機能です。
対応するファイルを開いたときに使えます。

| キー | 何が起きるか |
| --- | --- |
| `gd` | 定義へ移動する |
| `K` | カーソル下の説明を見る |
| `gi` | 実装へ移動する |
| `gr` | 参照箇所を見る |
| `<leader>rn` | 名前をまとめて変更する |
| `<leader>ca` | コードアクションを開く |
| `[d` | 前のエラー、警告へ移動する |
| `]d` | 次のエラー、警告へ移動する |
| `<leader>e` | エラー、警告の詳細を表示する |
| `<leader>q` | エラー、警告一覧を location list に入れる |
| `<leader>f` | 今のバッファを手動フォーマットする |

有効にしている主な LSP です。

| 言語・用途 | LSP |
| --- | --- |
| Lua | `lua_ls` |
| Go | `gopls` |
| Python | `pyright` |
| Rust | `rust_analyzer` |
| TypeScript / JavaScript | `vtsls` |
| ESLint | `eslint` |
| HTML | `html` |
| CSS / SCSS / Less | `cssls` |
| JSON / JSONC | `jsonls` |
| Tailwind CSS | `tailwindcss` |
| Emmet | `emmet_language_server` |

## Treesitter

Treesitter はコードの色分けや構文理解を強くするための仕組みです。

| コマンド | 何が起きるか |
| --- | --- |
| `:TSInstall <lang>` | 指定した言語の parser を入れる |
| `:TSUpdate` | インストール済み parser を更新する |

よく使うフロントエンド系の filetype に対応しています。
例: `typescript`、`typescriptreact`、`javascript`、`javascriptreact`、`html`、`css`、`scss`、`json`、`vue`、`svelte`、`astro`

## メモ

- この設定は、プロジェクト側に `.vscode/settings.json`、`.editorconfig`、formatter 設定などを自動作成しません。
- `biome.json`、`.prettierrc`、ESLint 設定などは、各プロジェクトに置く想定です。
- 保存時の自動フォーマットは無効です。必要なときだけ `<leader>f` で手動フォーマットします。
